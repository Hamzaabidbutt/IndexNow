IndexJet Documentation

Welcome! IndexJet gets your URLs in front of search engine crawlers through official APIs — the Google Indexing API, Bing URL Submission and the IndexNow protocol. This guide takes you from zero to your first indexed URL in under five minutes.

Quickstart

1. Create an account. Sign up free — no credit card required, 50 URL credits included.

2. Verify your site. Add your domain in the dashboard and verify ownership via DNS TXT record, HTML meta tag or file upload. Verification usually completes in under a minute.

3. Submit your first URL. Use the dashboard widget, or grab an API key from Dashboard → API Keys and run:

bash
curl -X POST https://api.indexjet.io/v1/submit \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"urls": ["https://yoursite.com/new-page"]}'

4. Track status. Watch the URL move through queued → processing → submitted → indexed in the dashboard, or poll GET /v1/submissions/:id.

Core Concepts

Projects

A project groups URLs, credentials and reports for one website (or one client, if you're an agency). Credits are pooled at the account level and can be allocated per project.

Credits

One credit = one unique URL submitted in a billing cycle. Duplicate submissions of the same canonical URL within 24 hours are free. Permanently failed submissions are refunded automatically.

Engines

Each submission fans out to every engine enabled for the project: Google (Indexing API), Bing (URL Submission API) and IndexNow participants (Bing, Yandex, Seznam.cz, Naver). Per-engine results appear in your reports.

Submitting a Single URL

From the dashboard, paste a URL into the submission box and press Index This URL. IndexJet will:

  1. Validate the URL syntax and resolve redirects
  2. Check robots.txt and the page's canonical tag
  3. Queue the URL with the right priority
  4. Submit to each enabled engine and record the response

Bulk Submission

Three ways to submit in bulk:

  • Paste — up to 10,000 URLs, one per line
  • Upload — .txt (one URL per line) or .csv (URLs in the first column), max 10 MB
  • API — POST /v1/submit accepts up to 1,000 URLs per request; loop for more

Every batch is deduplicated against your submission history before credits are consumed.

Sitemap Monitoring

Add your XML sitemap under Project → Sitemaps. IndexJet polls it on your chosen interval (15 min – 24 h), diffs it against the previous crawl and automatically submits new or updated URLs based on <lastmod>.

bash
curl -X POST https://api.indexjet.io/v1/sitemaps \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://yoursite.com/sitemap.xml", "interval": "1h"}'

Scheduled Indexing

Create schedules to submit URL sets at specific times — ideal for content calendars and product launches. Schedules support one-off runs, cron-style recurrence and timezone awareness.

Webhooks

Register a webhook endpoint to receive events in real time:

json
{
  "event": "url.indexed",
  "created_at": "2026-07-18T09:41:00Z",
  "data": {
    "url": "https://yoursite.com/new-page",
    "engine": "bing",
    "submission_id": "sub_9f8e7d6c",
    "latency_ms": 1380
  }
}

Events: url.queued, url.submitted, url.indexed, url.failed, credits.low. Every request is signed with an HMAC-SHA256 signature in the X-IndexJet-Signature header — always verify it.
