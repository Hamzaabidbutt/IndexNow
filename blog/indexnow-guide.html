The Complete IndexNow Guide

IndexNow is an open protocol, launched by Microsoft and Yandex in 2021, that lets websites instantly notify search engines when content is created, updated or deleted. One ping reaches every participating engine. Here's everything you need to implement it.

How It Works

  1. You generate an API key (any 8–128 character hex/alphanumeric string).
  2. You prove ownership by hosting the key at https://yoursite.com/<key>.txt.
  3. When a URL changes, you send it to any IndexNow endpoint with your key.
  4. The receiving engine shares the notification with all other participants.
bash
curl "https://api.indexnow.org/indexnow?url=https://yoursite.com/new-page&key=YOUR_KEY"

Bulk submission (up to 10,000 URLs per request) uses a POST body:

json
POST https://api.indexnow.org/indexnow
Content-Type: application/json

{
  "host": "yoursite.com",
  "key": "YOUR_KEY",
  "urlList": [
    "https://yoursite.com/new-page",
    "https://yoursite.com/updated-page"
  ]
}

Which Engines Participate?

BingGlobalMinutes to 1 hour

Google does not participate. Google discovers content via crawling, sitemaps and its own APIs — which is why a complete indexing strategy submits through multiple channels simultaneously.

Best Practices

  • Submit on real changes only. Pinging unchanged URLs repeatedly damages the trust engines place in your notifications.
  • Include deletions. Submitting removed URLs (now returning 404/410) gets them dropped from indexes faster.
  • Batch when possible. One POST with 500 URLs beats 500 GETs.
  • Keep the key file stable. Rotating keys is fine, but the old file must stay up until pending notifications settle.

Implementation Options

Popular CMS plugins (WordPress, Shopify apps) fire IndexNow pings on publish. For custom stacks, wire a ping into your deploy or publish pipeline. Or skip the plumbing entirely: IndexJet generates and hosts your key, batches notifications, dedupes, retries failures and reports per-engine outcomes — alongside Google submission in the same dashboard.
