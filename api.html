API Reference

The IndexJet REST API is organized around predictable resource URLs, returns JSON on every response, and uses standard HTTP status codes and verbs. Base URL:

text
https://api.indexjet.io/v1

Authentication

Authenticate with a Bearer token. Create and revoke keys under Dashboard → API Keys. Keys are scoped per project and shown only once at creation — store them securely and never commit them to source control.

bash
Authorization: Bearer ij_live_xxxxxxxxxxxxxxxxxxxx

POST/v1/submit

Submit up to 1,000 URLs for indexing. Requests are idempotent when an Idempotency-Key header is provided.

bash
curl -X POST https://api.indexjet.io/v1/submit \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: 4f9d2c1a-launch-batch-07" \
  -d '{
    "urls": [
      "https://yoursite.com/new-page",
      "https://yoursite.com/updated-page"
    ],
    "priority": "high",
    "engines": ["google", "bing", "indexnow"]
  }'

Response 202 Accepted:

json
{
  "batch_id": "bat_7d1e2f3a",
  "accepted": 2,
  "duplicates": 0,
  "invalid": 0,
  "credits_charged": 2,
  "submissions": [
    { "id": "sub_9f8e7d6c", "url": "https://yoursite.com/new-page", "status": "queued" },
    { "id": "sub_1a2b3c4d", "url": "https://yoursite.com/updated-page", "status": "queued" }
  ]
}

GET/v1/submissions/:id

Retrieve the live status and per-engine results of a single submission.

json
{
  "id": "sub_9f8e7d6c",
  "url": "https://yoursite.com/new-page",
  "status": "indexed",
  "created_at": "2026-07-18T09:40:12Z",
  "engines": {
    "google": { "status": "submitted", "submitted_at": "2026-07-18T09:40:14Z" },
    "bing": { "status": "indexed", "indexed_at": "2026-07-18T10:02:51Z" },
    "indexnow": { "status": "submitted", "submitted_at": "2026-07-18T09:40:13Z" }
  }
}

GET/v1/submissions

List submissions with cursor pagination and filters.

bash
curl "https://api.indexjet.io/v1/submissions?status=failed&limit=50&cursor=eyJpZCI6..." \
  -H "Authorization: Bearer YOUR_API_KEY"

Query parameters: status (queued | processing | submitted | indexed | failed), url (prefix match), from/to (ISO 8601), limit (max 100), cursor.

DELETE/v1/submissions/:id

Cancel a queued submission. Only submissions in queued state can be cancelled; the credit is returned to your balance.

GET/v1/account

Returns plan, credit balance and usage for the current billing period.

json
{
  "plan": "pro",
  "credits": { "included": 15000, "used": 8412, "remaining": 6588 },
  "period_ends": "2026-08-01T00:00:00Z",
  "rate_limit": { "requests_per_minute": 300 }
}

SDKs & Examples

Official libraries keep pagination, retries and idempotency handled for you:

javascript
import { IndexJet } from "@indexjet/sdk";

const client = new IndexJet(process.env.INDEXJET_API_KEY);

const batch = await client.submit({
  urls: ["https://yoursite.com/new-page"],
  priority: "high",
});

console.log(batch.submissions[0].status); // "queued"
python
from indexjet import IndexJet

client = IndexJet(api_key=os.environ["INDEXJET_API_KEY"])

batch = client.submit(urls=["https://yoursite.com/new-page"])
print(batch.submissions[0].status)  # "queued"

See the documentation for webhooks, rate limits and error handling.
